# AI, ML, Deep Learning, and Generative AI

These four terms get used loosely, but the exam expects you to know how they relate. They're **nested**, each one a subset of the one before.

## The nesting

- **Artificial Intelligence (AI)** - the broadest idea: machines performing tasks that normally need human intelligence (understanding language, recognising images, making decisions).
- **Machine Learning (ML)** - a subset of AI where systems **learn patterns from data** instead of being explicitly programmed with rules.
- **Deep Learning (DL)** - a subset of ML that uses **neural networks** with many layers. It powers image recognition, speech, and language, and needs lots of data and compute.
- **Generative AI** - a subset of deep learning that **creates new content** (text, images, audio, code) using large **foundation models**.

```
AI  ⊃  Machine Learning  ⊃  Deep Learning  ⊃  Generative AI
```

## Rules vs learning

Traditional software follows rules you write: "if the email contains X, flag it." ML instead **learns the rules from examples**. Show it thousands of spam and non-spam emails and it works out the patterns itself, and keeps improving as it sees more data.

## Three words you'll see constantly

- **Model** - the thing that has learned the patterns; you give it input and it produces output.
- **Training** - the process of feeding data to create or improve a model.
- **Inference** - using a trained model to make a prediction or generate output on new input.

## Everyday examples

| Concept | Example |
|---|---|
| AI | A thermostat that "decides" when to heat |
| ML | Netflix recommending shows from your viewing history |
| Deep learning | Face unlock on your phone |
| Generative AI | A chatbot writing an email draft for you |

## Key takeaways

- AI ⊃ ML ⊃ deep learning ⊃ generative AI, each is a narrower subset.
- ML learns patterns from data rather than following hand-written rules.
- Model = learned patterns; training = building it; inference = using it.
- Generative AI creates new content using large foundation models.
